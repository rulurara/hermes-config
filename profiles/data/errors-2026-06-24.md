# 📝 데이터 AI 오류 기록

## 1. 나라장터 (G2B) API 연계 실패

### 증상
- URL: `https://www.g2b.go.kr/ep/invitation/publish/list.do`
- HTTP 200 반환되지만 "요청하신 페이지를 찾을수 없습니다" (404 에러 페이지)
- 여러 URL 패턴 시도 모두 실패

### 원인
- 나라장터 사이트 구조 변경 (2026년 상반기)
- 기존 URL: `/ep/invitation/publish/list.do` → 접근 불가
- 오픈 API: `openapi.g2b.go.kr` → 서비스키 없이 호출 불가

### 시도한 것
1. `/ep/invitation/publish/list.do` → 404
2. `/front/invitation/publish/list.html` → 404
3. `/xe/search/Search.do` → 인코딩 오류
4. `openapi.g2b.go.kr/openapi/service/infobase/fclt/list` → 서비스키 필요

### 결론
- 나라장터 크롤링은 현재 불가능 (사이트 구조 변경)
- 대체 소스 필요: 공공데이터포털 API (서비스키 필요) 또는 다른 공공 API

## 2. GitHub API 데이터 품질 문제

### 증상
- 100건 수집 완료
- 이름/회사/자기소개 모두 0%

### 원인
- GitHub Search API가 기본 프로필 정보를 반환하지 않음
- `https://api.github.com/search/users` → login, id, avatar_url 등만 제공
- 상세 정보: `https://api.github.com/users/{username}` → 개별 호출 필요

### 결론
- 현재 수집 데이터는 login ID와 기본 메타데이터만 있음
- 상세 프로필 분석하려면 개별 사용자 API 호출 필요 (속도 제한 주의)

---

## 3. 환경 문제

### 증상
- `pip3` 명령어 없음
- `beautifulsoup4` 모듈 없음
- 공공데이터포털 사이트 인코딩 오류 (`utf-8' codec can't decode byte 0xc0`)

### 원인
- Ubuntu 서버에 pip3 미설치 → `python3 -m pip` 또는 `apt install python3-pip` 필요
- 공공데이터포털 사이트가 gzip 압축 → 브라우저에서도 디코딩 실패

### 결론
- 서버 파이프라인 구축 시 pip3 설치 필요
- 공공데이터는 API 사용 권장 (웹 크롤링 비권장)

| 순위 | 항목 | 난이도 | 비고 |
